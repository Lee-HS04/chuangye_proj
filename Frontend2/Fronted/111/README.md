# R2P-Guard - 运动员状态管理系统

## 项目简介

R2P-Guard 是一个苹果风格的未来科技感运动员状态管理系统，用于全方位管理运动员的赛前状态、训练准备和疲劳恢复。界面设计简约大气，具有良好的交互感。

## 核心功能

### 1. 赛前状态评估
- **状态分级**：通过红、黄、绿三色灯直观展示运动员状态
- **详细建议**：根据评估结果提供具体的行动建议
- **关键指标**：展示体能状态、肌肉状态、关节灵活度和整体状态等关键指标
- **状态超过平均水平**：整体状态超过100%的优秀表现
- **状态切换功能**：点击不同状态按钮时，蓝色选中状态会随之改变

### 2. 个体化运动准备
- **训练前状态评分**：评估运动员的训练前状态
- **个性化热身方案**：根据状态生成个性化的热身建议
- **热身视频播放**：依次播放热身动作示范视频
- **热身后复测**：完成热身后再次检测状态

### 3. 疲劳恢复与后续决策
- **疲劳恢复评估**：评估运动员的疲劳状态
- **恢复方案推荐**：提供针对性的恢复建议
- **恢复活动列表**：详细的恢复活动安排
- **身体状态趋势**：预留趋势分析功能（累计使用7天解锁）

### 4. 人体模型分析
- **交互式人体模型**：高级简约的人体模型，包含身体各个部位
- **部位选择**：点击身体部位查看详细信息
- **部位指标**：展示每个部位的状态、疲劳度、灵活性和力量等指标
- **部位分析**：对选中部位进行详细分析

### 5. 用户管理系统
- **用户注册**：支持邮箱注册，包含SMTP邮件验证码功能
- **用户登录**：用户名和密码登录
- **个人信息收集**：收集用户的年龄、性别、运动时间和身份信息
- **个人信息修改**：支持修改已填写的个人信息
- **数据存储**：使用localStorage存储用户数据

### 6. 个人信息收集
- **年龄**：<= 18岁、18 ~ 22岁、22 ~ 26岁、26 ~ 30岁、30 ~ 34岁、34 ~ 38岁、38 ~ 42岁、>= 42岁
- **性别**：男、女
- **一周运动几小时**：基本不运动、<= 2小时、2小时 ~ 10小时、>= 10小时
- **身份**：业余爱好者、退役运动员、运动员、学生、教练

## 完整流程

1. **欢迎页**：打开系统，查看轮播介绍，点击"完成"按钮进入登录页面
2. **登录/注册**：
   - 新用户：注册账户（需要邮箱、邮箱验证码、用户名、密码）
   - 老用户：直接登录（使用用户名和密码）
3. **个人信息收集**：
   - 首次登录：填写个人信息（年龄、性别、一周运动几小时、身份）
   - 后续登录：如果已填写个人信息，直接进入主页面
4. **主页面**：
   - 点击蓝色"开始检测"按钮
   - 点击右上角用户头像查看个人信息
   - 点击"修改个人信息"按钮修改已填写的信息
5. **测试类型选择**：选择赛前状态评估、个体化运动准备或疲劳恢复与决策
6. **动作示范视频**：观看相应的动作示范视频，注意查看视频下方的注意事项
7. **321倒计时**：准备开始动作
8. **摄像头拍摄**：进行动作拍摄
9. **视频预览**：查看拍摄的视频，选择上传或重新拍摄
10. **结果展示**：查看评估结果和建议

## 如何使用

### 方法一：使用本地服务器（推荐）
1. 进入 `frontend` 目录
2. 启动本地服务器：
   ```bash
   # 使用Python 3
   python3 -m http.server 8002
   
   # 或使用Python 2
   python -m SimpleHTTPServer 8002
   ```
3. 打开浏览器，访问 `http://localhost:8002/111/welcome/app_welcome.html`
4. 按照完整流程进行操作

### 方法二：直接打开文件
1. 直接在浏览器中打开 `111/welcome/app_welcome.html` 文件
2. 查看轮播介绍，点击第三张图片进入登录页面
3. 注册新账户或登录现有账户
4. 填写个人信息
5. 进入主功能页面，选择要进行的测试类型
6. 按照页面提示完成整个检测流程
7. 查看评估结果和建议

### 验证码功能
- 注册时，点击"发送验证码"按钮
- 验证码会通过SMTP邮件发送到您的邮箱
- 输入收到的验证码完成注册

### 邮箱服务配置
如需使用邮箱验证码功能，需要配置邮件发送服务：

1. 编辑 `frontend/email_server.py` 文件中的邮件配置：
   ```python
   EMAIL_CONFIG = {
       'smtp_server': 'smtp.gmail.com',  # SMTP服务器地址
       'smtp_port': 587,  # SMTP端口
       'sender_email': 'your_email@gmail.com',  # 发件人邮箱
       'sender_password': 'your_app_password',  # 邮箱授权码/密码
   }
   ```

2. 启动邮箱服务：
   ```bash
   python email_server.py
   ```

3. 服务运行在 `http://127.0.0.1:8001`
